## 1. 홈 페이지 전면 개편

### 변경 내용

기존 `src/app/page.tsx`는 단순히 `/products`로 즉시 리다이렉트하는 코드만 존재했습니다.

```tsx
// 기존 — 아무런 맥락 없이 즉시 이동
export default function RootPage() {
  redirect('/products')
}
```

이를 다음과 같이 구성된 완전한 홈 페이지로 대체했습니다:

| 섹션 | 역할 |
|------|------|
| **HeroSection** | KU 엠블럼, 브랜드 헤드카피, 주요 CTA 버튼 |
| **QuickActionGrid** | 핵심 기능 4가지 바로가기 카드 |
| **RecentProducts** | 최근 등록 상품 6개 (서버 컴포넌트, Supabase 직접 조회) |
| **RecommendBanner** | AI 추천 기능 2종 하이라이트 |

### UX 이론 근거

#### 1-1. 맥락적 착지 (Contextual Landing) — 첫 인상의 법칙

Nielsen Norman Group의 연구에 따르면, 사용자는 **페이지에서 평균 10–20초 내에 머물지 이탈할지 결정**합니다. 기존 즉시 리다이렉트는 서비스 정체성을 전달하는 기회를 완전히 버리는 설계입니다.

홈 페이지를 두는 것 자체가 **첫 인상(First Impression)**을 설계하는 행위입니다. 고려대학교 엠블럼과 버건디 색상을 전면에 배치함으로써:
- 서비스가 고려대 커뮤니티에 특화되었음을 즉시 전달
- 신뢰도(Credibility) 신호 제공 — 학교 아이덴티티 == 검증된 커뮤니티

#### 1-2. 멘탈 모델 (Mental Model)

Don Norman의 *The Design of Everyday Things*에서 정의한 멘탈 모델은 "사용자가 시스템이 어떻게 작동하는지에 대해 가진 내적 표상"입니다.

사용자들은 당근마켓, 번개장터 같은 중고거래 플랫폼의 멘탈 모델을 이미 보유하고 있습니다:

> "홈 화면 → 최근 상품 목록 → 상품 상세"

**RecentProducts** 섹션을 홈에 배치한 것은 이 기존 멘탈 모델에 부합합니다. 사용자가 새로운 학습 없이 즉시 탐색을 시작할 수 있습니다.

#### 1-3. Progressive Disclosure (점진적 공개)

Bruce Tognazzini가 정립한 원칙: **처음에는 필수 정보만, 세부사항은 사용자가 요청할 때** 보여준다.

홈 페이지는 4가지 QuickAction만 노출하고, 나머지 기능들(채팅, 위시리스트, 마이페이지 등)은 NavBar로 숨깁니다. 이는 인지 부하(Cognitive Load)를 낮추고, 처음 방문한 사용자도 "무엇을 할 수 있는지" 쉽게 파악하게 합니다.

#### 1-4. F-Pattern 스캔 패턴

Nielsen의 Eye-tracking 연구: 사용자는 웹 페이지를 F자 모양으로 스캔합니다.

- 상단 HeroSection → 주목도 최대 (브랜드, CTA)
- 중단 QuickActionGrid → 핵심 기능 탐색
- 하단 RecentProducts, RecommendBanner → 추가 탐색 의향 있는 사용자

콘텐츠를 위에서 아래로 중요도 순으로 배치했습니다.

## 2. 네비게이션 바 개선

### 변경 내용

```tsx
// 추가된 항목
{ label: '패키지 추천', href: '/recommend/package' },
{ label: '시세 추천',   href: '/recommend/resale-price' },
```

기존 NavBar에는 상품/패키지/채팅/위시리스트/상품등록/마이페이지만 있었고, 추천 기능이 숨겨져 있었습니다.

### UX 이론 근거

#### 2-1. Visibility of System Status — Nielsen의 10가지 휴리스틱 #1

**"시스템은 항상 합리적인 시간 내에 적절한 피드백을 통해 현재 무슨 일이 일어나고 있는지 사용자에게 알려야 한다."**

더 넓게 적용하면: **기능이 존재한다는 사실 자체도 사용자에게 '보여야' 한다**. 내비게이션에 없는 기능은 사실상 존재하지 않는 것과 같습니다.

추천 링크를 NavBar에 추가함으로써:
- 사용자가 AI 추천 기능을 발견할 수 있게 됨 (Discoverability)
- 학교 커뮤니티 특화 기능임을 상시 노출

#### 2-2. Miller의 법칙과 청킹 (Chunking)

George A. Miller의 연구: 인간의 단기 기억은 **7±2 개의 항목**을 처리하는 데 최적화되어 있습니다.

현재 NavBar에는 8개 항목이 있습니다. 이는 임계치에 가깝지만, 항목들이 의미적으로 유사한 그룹으로 나뉘기 때문에(상품 관련 / 추천 관련 / 커뮤니티 관련 / 내 계정 관련) 실제 인지 부담은 낮습니다.

향후 NavBar가 더 복잡해진다면, 추천 메뉴를 드롭다운으로 통합하는 것이 권장됩니다.

## 3. 패키지 추천 — 카드 선택 및 합계 금액

### 변경 내용

- 패키지 추천 결과 카드를 클릭하면 선택/해제 토글
- 선택된 상품들의 합계 금액이 실시간으로 표시되는 고정 바(Sticky Bar) 추가
- 외부 링크 아이콘을 통해 원 상품 페이지로 이동 가능

### UX 이론 근거

#### 3-1. Affordance (행동 유도성)

Don Norman이 정의한 Affordance: **객체의 물리적 속성이 그것을 어떻게 사용할 수 있는지 암시**해야 합니다.

기존 카드는 `<Link>` 요소였기 때문에 "클릭하면 상품 페이지로 이동한다"는 Affordance만 있었습니다. 이를 **선택 가능한 체크박스 카드**로 바꿈으로써 "선택/취합한다"는 새로운 Affordance를 추가했습니다.

원형 체크박스(선택 시 파란색 채워짐)는 사용자가 **이미 알고 있는 체크박스 패턴**을 재활용하여 학습 비용 0으로 인터랙션을 이해하게 합니다.

#### 3-2. Instant Feedback — 즉각적 피드백의 원칙

사용자가 카드를 클릭하는 순간 합계 금액이 업데이트됩니다. Nielsen 휴리스틱 #1(Visibility of System Status)의 적용:

> 피드백 지연이 1초를 넘으면 사용자는 자신의 행동이 효과가 있었는지 의심하기 시작한다. — Jakob Nielsen

`useState<Map<string, number>>`로 클라이언트 상태를 관리하고, `Array.from(selectedItems.values()).reduce(...)` 계산을 렌더 시마다 동기적으로 실행하기 때문에 클릭 → 합계 표시까지의 지연이 0에 수렴합니다.

#### 3-3. Goal-Gradient Effect (목표 구배 효과)

심리학에서: **목표에 가까워질수록 행동이 가속화**됩니다.

합계 금액 바는 "총 얼마를 쓸 것인가"를 실시간으로 시각화합니다. 이는 사용자가 자신의 예산 목표에 얼마나 근접했는지를 인식하게 하여, 구매 결정에 대한 확신을 높입니다.

## 4. 시세 추천 — 구매가 입력 추가

### 변경 내용

재판매 가격 추천 폼에 **구매가(원래 산 가격)** 선택 입력 필드 추가.
- 미입력 시: 카테고리 평균 거래가 기준으로 감가율 계산
- 입력 시: 내 구매가 기준으로 계산 후, 카테고리 평균은 참고 정보로 표시

### UX 이론 근거

#### 4-1. User Control and Freedom — Nielsen 휴리스틱 #3

**"사용자는 원치 않는 상태에서 벗어날 수 있는 명확한 '비상구'가 필요하다."**

더 넓게는, 사용자에게 **맥락에 맞는 선택권**을 주어야 합니다. 구매가를 기억하는 사용자와 모르는 사용자 모두를 수용하는 인터페이스가 더 포용적입니다.

- 구매가를 아는 사용자: 더 정확한 개인화 결과 제공
- 구매가를 모르는 사용자: 시장 평균 기준으로 fallback → 어떤 경우에도 답을 제공

#### 4-2. Personalization과 Relevance (개인화와 관련성)

**"나의 상황에 딱 맞는 결과"**는 사용자 신뢰도와 만족도를 높입니다.

카테고리 평균 기반 결과는 **일반적인 시세**를, 내 구매가 기반 결과는 **내 물건의 예상 감가액**을 알려줍니다. 이 차이는 판매자 입장에서 매우 중요하며, 개인화된 데이터는 더 실행 가능한(actionable) 인사이트를 제공합니다.

#### 4-3. Transparency (투명성)

결과 화면에서 "내 구매가", "카테고리 평균 거래가 참고" 두 값을 모두 표시함으로써:

- 사용자가 계산 방식을 이해할 수 있음 → **신뢰도 상승**
- 알고리즘의 블랙박스화를 방지 → Explainable UX

## 5. 인증 미들웨어 활성화 및 next 파라미터

### 변경 내용

`src/proxy.ts`에 `?next=` 파라미터 보존 로직 추가:

```ts
// 변경 전
url.pathname = '/login'
return NextResponse.redirect(url)

// 변경 후
url.pathname = '/login'
const nextParam = pathname + (request.nextUrl.search || '')
url.searchParams.set('next', nextParam)
return NextResponse.redirect(url)
```

### UX 이론 근거

#### 5-1. Consistency and Standards — Nielsen 휴리스틱 #4

사용자는 웹 전반에서 **로그인 후 원래 페이지로 돌아가는 패턴**을 학습했습니다(Gmail, GitHub, 네이버 등). 이 기대에 부응하지 않으면 불필요한 마찰이 생깁니다.

기존 구현은 모든 미인증 접근 시 `/products`로 귀결되었습니다. 사용자가 `/wishlist`를 북마크해뒀거나, 채팅 링크를 공유받은 경우, 로그인 후 목적지를 잃어버리는 **탐색 좌절(Navigation Frustration)**이 발생합니다.

#### 5-2. Task Completion Rate (태스크 완료율)

UX 연구에서 태스크 완료율은 핵심 지표입니다. `?next=` 파라미터가 없으면:

1. 사용자가 상품 A 페이지를 공유받음
2. 로그인 화면으로 리다이렉트
3. 로그인 완료 → `/products` 이동 (상품 A가 아님)
4. 사용자가 상품 A를 다시 검색해야 함

태스크(상품 A 보기)를 완료하기 위한 **추가 단계(Extra Steps)**가 발생합니다. `?next=` 보존으로 이 마찰을 제거합니다.

## 6. KU 버건디 디자인 시스템

### 색상 팔레트

| 역할 | 색상 코드 | 사용처 |
|------|-----------|--------|
| Primary | `#8B0029` | HeroSection 배경, 주요 CTA, 링크 호버 |
| Primary Dark | `#6B0020` | 그라디언트 끝색, 오버레이 |
| Accent Background | `#FDF2F4` | QuickAction 아이콘 배경 |
| Warm Background | `#FAF9F7` | (선택적) 섹션 배경 |

### UX 이론 근거

#### 6-1. Color as Communication (색상의 의미 전달)

색상 이론에서 색상은 **의미와 감정**을 전달합니다. 버건디(짙은 빨강)는:
- **권위, 전통, 신뢰**의 감정을 환기
- 고려대학교 아이덴티티 색상 → **브랜드 일관성(Brand Consistency)**
- 파란색 계열(인디고)에서 변경함으로써 플랫폼 전용 아이덴티티 구축

#### 6-2. Contrast and Accessibility (대비와 접근성)

`#8B0029` (버건디) 위에 흰색 텍스트의 명도 대비비는 약 **7.3:1**로 WCAG 2.1 AA 기준(4.5:1)을 충족합니다. 시각적 접근성을 확보하면서도 브랜드 색상을 유지합니다.

#### 6-3. Gestalt 원칙 — 근접성(Proximity)과 유사성(Similarity)

홈 페이지에서 버건디 색상은 **행동 가능한 요소(CTA, 링크, 강조)**에 일관되게 사용됩니다. 이는 Gestalt 유사성 원칙에 따라 "같은 색 = 같은 종류의 요소"임을 인지적으로 학습하게 합니다. 사용자가 페이지를 탐색할 때 버건디를 보면 "여기서 행동할 수 있다"고 자동으로 인식하게 됩니다.
